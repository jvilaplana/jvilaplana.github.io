Shell script

  • Shell script is a series of command written in a plain text file
  • Why write shell scripts?
    • Shell scripts can take input from a user or file and output to screen
    • Useful to create our own commands
    • Save lots of time
    • Automate some everyday tasks
    • System administration part can be also automated
  • Command types
    • Internal: Commands included in the shell (cd, pwd, set, ...)
    • External: Programs not included in the shell (cp, chown, ls cut, ...)

External commands location

  • External commands are searched in the directories specified by the PATH environment variable
root@winterfell:~# echo $PATH
/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/usr/games:/usr/local/games
  • To modify the search path:
root@winterfell:~# PATH=~/bin/:$PATH

Environment variables

  • There are some environment variables that can have an impact to the command-line interpreter
  • The main ones are:
    • PATH - routes to search external commands
    • HOME - home directory of current user
    • USER - current user
    • PWD - current directory
    • MAIL - file where mail is stored
  • The default values of these variables can be modified in the startup files of the command-line interpreter

What is a login or non-login shell?

When you login (type username and password) via console, either sitting at the machine, or remotely via ssh: .bash_profile is executed to configure your shell before the initial command prompt.

But, if you’ve already logged into your machine and open a new terminal window (xterm) inside Gnome or KDE, then .bashrc is executed before the window command prompt. .bashrc is also run when you start a new bash instance by typing /bin/bash in a terminal.

Startup shell files

  • These files prepare the execution environment that each users needs to use the system
  • Startup configuration files (login shell):
    • /etc/profile
    • ~/.bash_profile
    • ~/.bash_login
    • ~/.profile
  • Startup configuration files (non-login shell):
    • ~/.bashrc
  • Logout configuration files:
    • ~/.bash_logout

Script programming

  • Most interpreters provide their own programming language that allows the creation of programs (shell-scripts)
  • These scripts can invoke any command available in interactive mode
  • There are additional commands designed to support the programming inside the shell (variables, control structures, functions, ...)
  • Script files are plain text files with execution permission:
    chmod 755 my_script or chmod +x my_script
  • The first line of the script is used to specify the location of the command-line interpreter that will be used:
    #!/bin/bash
  • The execution of scripts is done in the same way as regular executables: ./my_script

Variables

  • Bash allows the declaration and usage of variables, defined as character strings
  • There are three types of variables:
    • Environment variables (editable, read-only)
    • User-defined variables
    • Read-only variables (constants)

Read-only environment variables

  • These variables can be accessed but not modified
  • Relative to command-line arguments:
    • $0 - name of the script
    • $1-$9 - input arguments 1-9
    • ${10},${11},... - input arguments > 9
    • $* - all the arguments
    • $@ - all the arguments
    • $# - number of arguments
  • Relative to the process:
    • $$ - current process id (script)
    • $! - most recent background process id
    • $? - most recent command exit status

Example

#!/bin/bash
echo "Script name: $0"
echo "Number of arguments: $#"
echo "Arguments are: $@"
ls -lh
echo "Last command exit code: $?"
more wololo
echo "Last command exit code: $?"
echo "My PID: $$"
top &
echo "Last command PID: $!"
kill -9 $!
echo "Last command exit code: $?"
exit 0

User-defined variables

  • Created and initialized by the user
  • Their scope is restricted to the shell. Using export they can be made accessible to any subsequent process that runs in the same shell process
  • No need to declare a variable
  • A variable can contain a number, a character or a string of characters
#!/bin/bash
myname=Arya
echo myname
echo $myname
myname="Arya       Stark"
echo $myname
echo "$myname"
echo '$myname'

Lists

  • All elements within the list have the same type
  • The first element has the position 0
  • There is not a defined maximum size
  • It is not necessary to define all the indexes
#!/bin/bash
mylist=(Winter is comming)
echo ${mylist[0]}
echo ${mylist[1]}
echo ${mylist[2]}
mylist[10]=run
echo ${mylist[10]}
echo ${mylist[*]}         # All of the items in the array
echo ${!mylist[*]}        # All of the indexes in the array
echo ${#mylist[*]}        # Number of items in the array
echo ${#mylist[0]}        # Length of item zero

  • It can be done using backquotes (`...`) or using $()

Exec command

  • Equivalent to the execl system call. Overlaps the code of the specified command over the process that is being executed
  • Once the exec command is completed, it is not possible to go back to the original process. Therefore the rest of the code is never executed
#!/bin/bash
echo "Script $0 started"
exec date
echo "Script $0 finished"       # This line will not be executed

Arithmetic operations

  • Variable values are stored as strings. To perform arithmetic or logic operations they must be transformed to integers, perform the operation and transform them back to string
  • There are multiple ways to perform this process:
    • let command
    • Expansion $((<expression>))
    • expr command

Arithmetic operators

  • +, - → Plus and minus
  • *, /, % → Multiplication, division and mod
  • ** → Exponentiation
  • <<, <<=, >>, >>=, ~, !, &, | → Bitwise operators
  • >=, <=, >, < → Comparators bigger and smaller
  • ==, != → Comparators equal and not equal
  • &&, || → Logical AND and OR

  • Carries out arithmetic operations on variables. Returns 0 when args are not evaluated to 0, 1 otherwise.

String manipulation

  • ${#string} Length of $string
  • ${string:pos} Extracts the substring starting at pos
  • ${string:pos:length} Extracts a substring of length characters starting at pos
  • ${string#pattern} Strip shortest match of $pattern from front of $string
  • ${string##pattern} Strip longest match of $pattern from front of $string
  • ${string%pattern} Strip shortest match of $pattern from back of $string
  • ${string%%pattern} Strip longest match of $pattern from back of $string
  • ${string/substring/replacement} Replace first match of $substring with $replacement
  • ${string//substring/replacement} Replace all matches of $substring with $replacement

The set command

  • Assigns the arguments of set to the values of command-line arguments ($1-$9)
#!/bin/bash
set hell to highway $(date)
echo $3 $2 $1
echo $@

Flow control structures

  • Bash includes the following flow control structures:
    • if then
    • case
    • for
    • while
    • until

The if then structure

  • Allows to perform a conditional execution depending on the value of a conditional expression
if <conditional_expr>
    then
    commands
fi
  • The conditional expression is evaluated from the result of a command or by using the test command

The test command

  • Allows to evaluate a condition, returning true (0) or false (1)
  • The following formats can be used:
test <expression>
[<expression>]
  • Allows to compare conditions about integers, strings, files or several logical expressions

Comparison operators: files

  • -e file file exists
  • -r file file is readable
  • -w file file is writable
  • -x file file is executable
  • -f file file is a regular file
  • -d file file is a directory
  • -h,-L file file is a symbolic link
  • -c file file is a character device
  • -b file file is a block device
  • -p file file is a pipe

Further detail about comparison file operators

Comparison operators: strings and numbers

  • Strings:
    • str str is not a null string
    • -z str str length is 0
    • -n str str length is not 0
    • str = str2 str and str2 are equal
    • str != str2 str and str2 are different
  • Numbers:
    • n1 -eq n2 n1 and n2 are equal
    • n1 -ne n2 n1 and n2 are different
    • n1 -gt n2 n1 is bigger than n2
    • n1 -ge n2 n1 is bigger or equal than n2
    • n1 -lt n2 n1 is smaller than n2
    • n1 -le n2 n1 is smaller or equal than n2

Comparison operators

  • Using the following operators more complex expressions can be built:
    • ! negation
    • (condition) to group expressions
    • -a logic AND operator
    • -o logic OR operator
["$n1" -gt "n2" -a -z "$str"]   # n1 > n2 and str is not empty
[! -n "$str"]                   # str is empty
[-f f1 -a -w f1]                # f1 is a file and can be written
[f1 -nt f2]                     # f1 is newer than f2
[f1 -ot f2]                     # f1 is older than f2
[f1 -ef f2]                     # f1 and f2 are links to the same file

Conditional structures

  • if else, if elif
if <expression>
    then
    commands
elif <expression>
    then
    commands
else
    commands
fi

Example

#!/bin/bash
if test $# -lt 2
  then # Input parameters check
  echo "Introduce at least 2 parameters"

  then # Check if it is the same file

case structure

  • Allows to implement a multiple conditional flux similar to multiple nested if structures
case $test-var in
  pattern1)
    commands
    ;;
  pattern2)
    commands
    ;;
  pattern3)
    commands
    ;;

case structure patterns

  • The patterns for each case option follow the same rules as file patterns
  • The following wildcards are available:
    • * any character or string
    • ? any single character
    • [...] a range of values
    • | alternative characters or strings (equivalent to the or operator)
[0-9]|[a-f]|[A-F]   # Hex digit
file*.txt           # Names starting with file and ending with .txt

case example

  d|D) date

  l|L) ls
    ;;
  q|Q) exit 0
    ;;
  *) echo "Incorrect option"
    exit 1
    ;;
esac

for structure

  • The for structure allows to implement a loop through all the words of a list
  • Syntax 1:

done

  • Syntax 2:

  commands

  echo "you will respawn in $i..."

}

Lists of commands

  • It is possible to execute multiple linked commands. The execution of the next command in the list depends on the result returned by the previous ones and the used list type
  • and commands lists:
    • The next command is executed if, and only if, the previous command returns an exit status of 0
    • Syntax: cmd1 && cmd2 && ... && cmdN
  • or commands lists:
    • The next command is executed if, and only if, the previous command returns an exit status different from 0
    • Syntax: cmd1 || cmd2 || ... || cmdN

Lists of commands example 1

#!/bin/bash
if [ ! -z "$1" ] && echo "arg1 = $1" && \
   [ ! -z "$2" ] && echo "arg2 = $2"
then
  echo "There are at least 2 arguments"
else
  echo "There are less than 2 arguments"

[ "$1" == "a" ] && echo "arg1 correct"
[ -f "$2" ] || echo "File $2 doesn't exist"
[ -f "$2" ] && ( tar -zcvf $2$(date + %F).tar.gz $2 ) && echo "File $2 compressed"

Debugging scripts

  • Shell scripts can be debugged using the -v or -x bash options
  • -v: Shows each line of the script before its execution and its result
  • -x: Shows each line of the script after the variable replacement but before its execution, and it also shows the result of the execution
bash -v script_23.sh arg1 arg2
      
bash -x script_23.sh arg1 arg2

Redirections

  • It is allowed to use redirection operators to redirect the standard input, output and error of the commands used within the script
  • Descriptor 1 represents the stdin and descriptor 2 the stderr
  • Redirection operators:
    • >, >> Redirect stdout to a file
    • < Redirect file to stdin
    • 2>, 2>> Redirect stderr to a file
#!/bin/bash
ls -lh > listing.txt                              # stdout to file
cat < script_23.sh                                # file to stdin
grep date * 2> err.log                            # stderr to a file
grep date * 1>&2                                  # stdout to stderr
cat "$(find ./ -name file_names)" 2> /dev/null    # shows files and delete stderr

Regular expressions

  • There are the following metacharacters:
    • ^ Match with the line start
    • $ Match with the line end
    • . Match any single character
    • * Preceding item will be matched 0 or more times
    • ? Preceding item is optional and will be matched, at most, once
    • {N} Preceding item is matched exactly N times
    • {N,} Preceding item is matched N or more times
    • {N,M} Preceding item is matched from N to M times
    • [...] Match with any of the characters included in the brackets
    • [^...] Match with any character not included in the brackets
    • \< Match with word start
    • \> Match with word end

Regular expressions example

#!/bin/bash
grep -nv '^$' $0          # Get the lines (with line number) that are not blank

ls -lh | grep '^d'        # List only directories

ls -lh | grep '[^\.sh]$'  # List all files not ending in .sh

grep '[a-z]\{5\}' $0      # List all lines with at least 5 consecutive lowercase letters

exit 0
