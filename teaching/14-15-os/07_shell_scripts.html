Shell script

  • Shell script is series of command written in plain text file
  • Why to write Shell Script?
    • Shell script can take input from user, file and output them on screen
    • Useful to create our own commands
    • Save lots of time
    • To automate some task of day today life
    • System Administration part can be also automated
  • Command types
    • Internal: Commands included in the shell (cd, pwd, set, ...)
    • External: Programs not included in the shell (cp, chown, ls cut, ...)

External commands location

  • External commands are searched in the directories specified by the PATH environment variable
root@winterfell:~# echo $PATH
/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/usr/games:/usr/local/games
  • To modify the search path:
root@winterfell:~# PATH=~/bin/:$PATH

Environment variables

  • There are some environment variables that can have an impact to the command-line interpreter
  • The main ones are:
    • PATH - routes to search external commands
    • HOME - home directory of current user
    • USER - current user
    • PWD - current directory
    • MAIL - file where mail is stored
  • The default values of these variables can be modified in the startup files of the command-line interpreter

What is a login or non-login shell?

When you login (type username and password) via console, either sitting at the machine, or remotely via ssh: .bash_profile is executed to configure your shell before the initial command prompt.

But, if you’ve already logged into your machine and open a new terminal window (xterm) inside Gnome or KDE, then .bashrc is executed before the window command prompt. .bashrc is also run when you start a new bash instance by typing /bin/bash in a terminal.

Startup shell files

  • These files prepare the execution environment that each users needs to use the system
  • Startup configuration files (login shell):
    • /etc/profile
    • ~/.bash_profile
    • ~/.bash_login
    • ~/.profile
  • Startup configuration files (non-login shell):
    • ~/.bashrc
  • Logout configuration files:
    • ~/.bash_logout

Script programming

  • Most interpreters provide their own programming language that allows the creation of programs (shell-scripts)
  • These scripts can invoke any command available in interactive mode
  • There are additional commands designed to support the programming inside the shell (variables, control structures, functions, ...)
  • Script files are plain text files with execution permission:
    chmod 755 my_script or chmod +x my_script
  • The first line of the script is used to specify the location of the command-line interpreter that will be used:
    #!/bin/bash
  • The execution of scripts is done in the same way as regular executables: ./my_script

Variables

  • Bash allows the declaration and usage of variables, defined as character strings
  • There are three types of variables:
    • Environment variables (editable, read-only)
    • User-defined variables
    • Read-only variables (constants)

Read-only environment variables

  • These variables can be accessed but not modified
  • Relative to command-line arguments:
    • $0 - name of the script
    • $1-$9 - input arguments 1-9
    • ${10},${11},... - input arguments > 9
    • $* - all the arguments
    • $@ - all the arguments
    • $# - number of arguments
  • Relative to the process:
    • $$ - current process id (script)
    • $! - most recent background process id
    • $? - most recent command exit status

Example

#!/bin/bash
echo "Script name: $0"
echo "Number of arguments: $#"
echo "Arguments are: $@"
ls -lh
echo "Last command exit code: $?"
more wololo
echo "Last command exit code: $?"
echo "My PID: $$"
top &
echo "Last command PID: $!"
kill -9 $!
echo "Last command exit code: $?"
exit 0

Double-quotes, technically, the expansion of $var is seen as one word. Without them, each element is seen as a different word

User-defined variables

  • Created and initialized by the user
  • Their scope is restricted to the shell. Using export they can be made accessible to any subsequent process that runs in the same shell process
  • No need to declare a variable
  • A variable can contain a number, a character or a string of characters
#!/bin/bash
myname=Arya
echo myname
echo $myname
myname="Arya       Stark"
echo $myname
echo "$myname"
echo '$myname'

Typing variables

  • declare -r var=1 --> Read only variable
  • declare -i var=1 --> Integer variable
#!/bin/bash
touch data1
touch data2
files=data*
echo $files
echo "$files"
num=666
echo $num
declare -i x=999
echo $x
x="nine"
echo $x

Deleting and listing variables

  • Declared variables exist while the shell is active
  • A variable can be deleted by assigning a null value to it or using the unset command
  • To list all defined variables in a script the commands set and env can be used
#!/bin/bash
name=Arya
echo $name
unset name
echo $name
name=Stark
echo $name
name=
echo $name
set

Lists

  • All elements within the list have the same type
  • The first element has the position 0
  • There is not a defined maximum size
  • It is not necessary to define all the indexes
#!/bin/bash
mylist=(Winter is comming)
echo ${mylist[0]}
echo ${mylist[1]}
echo ${mylist[2]}
mylist[10]=run
echo ${mylist[10]}
${mylist[*]}         # All of the items in the array
${!mylist[*]}        # All of the indexes in the array
${#mylist[*]}        # Number of items in the array
${#mylist[0]}        # Length of item zero
